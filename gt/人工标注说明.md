# Penn Action 人工标注说明

## 1. 当前类别比例（用于预期管理）

- 全量 ROI 伪标签（train+val+test）：
  - `push_up=226`
  - `jumping_jack=53`
  - 比例约 `4.26:1`
- 当前 15% 人工标注子集（`gt` 三个文件合并）：
  - `push_up=31`
  - `jumping_jack=9`
  - 比例约 `3.44:1`

## 2. 每个人需要标注哪个文件

- `gt/syq.csv`
- `gt/szh.csv`
- `gt/lrx.csv`

每人只修改自己的 CSV。

## 3. 如何根据 CSV 找到对应视频

CSV 的 `video_path` 形如：

- `dataset/JumpingJack/1061.avi`
- `dataset/PushUps/1351.avi`

其中视频 ID 分别是 `1061`、`1351`。  
在 Penn Action 原始数据中对应路径：

- 帧序列：`/data1/shiyuqi/Penn_Action/frames/<video_id>/`
- 官方标签：`/data1/shiyuqi/Penn_Action/labels/<video_id>.mat`

例如 `1061`：

- `/data1/shiyuqi/Penn_Action/frames/1061/`
- `/data1/shiyuqi/Penn_Action/labels/1061.mat`

## 4. 逐行标注时要改哪些列

建议按行修正，每行代表一个动作片段：

- 必改：
  - `start_time`
  - `end_time`
  - `label_name`
- 建议改：
  - `source_type` 统一改为 `manual_refined`
- 其余列（如 `assignee/split/video_path/source_file/label_id`）保留。

## 5. 一个视频多个动作怎么处理

- 允许同一 `video_path` 出现多行（多个动作片段）。
- 如果当前行不够，复制一行再修改时间段即可。
- 明显误检行可以删除。

## 6. 标注规则（必须遵守）

- 时间单位：秒
- 约束：`0 <= start_time < end_time`
- 类别仅使用：
  - `push_up`
  - `jumping_jack`
- 边界尽量贴近动作真实开始/结束帧。

## 7. 标注完成后

把三份 CSV（`syq/szh/lrx`）交回后，统一进行：

- 合并
- 去重
- 一致性校验
- 导出最终训练标注
