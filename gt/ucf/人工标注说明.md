# UCF 人工标注说明（帧级）

不要用播放器进度条直接估时间，误差会很大。  
这里改成和 Penn 一样的帧级标注流程。

## 1. 已生成的文件

- `mapping_video_to_annotator.csv`：每人负责的视频
- `syq_frame.csv`
- `szh_frame.csv`
- `lrx_frame.csv`
- `outputs/ucf_frame_labeling/frames/<video_key>/000001.jpg ...`

## 2. 每个人怎么标

1. 打开自己的 `*_frame.csv`。
2. 到该行 `frames_dir` 对应目录逐帧看图，确定动作开始帧与结束帧。
3. 填写 `start_frame`、`end_frame`（从 1 开始计数）。
4. `class_name` 保持 `push_up` 或 `jumping_jack` 即可。

## 3. 转回训练用时间标注

完成后执行：

```bash
conda run -n yolo python scripts/convert_ucf_frame_labels.py \
  --frame-csvs gt/ucf/syq_frame.csv gt/ucf/szh_frame.csv gt/ucf/lrx_frame.csv \
  --out-dir gt/ucf
```

脚本会自动把帧号换算为秒，并输出：

- `gt/ucf/syq.csv`
- `gt/ucf/szh.csv`
- `gt/ucf/lrx.csv`

## 4. 换算规则

- `start_time = (start_frame - 1) / fps`
- `end_time = end_frame / fps`
- 自动限制到视频总帧范围内
